Animate Plugin

Delay the “close” side of a toggle until its CSS finishes. The examples below demonstrate transition-based targets, keyframe-based loops, and even trigger-side animations using nothing but data attributes.

Transition-based target fade

The target removes hidden, waits one frame, adds panel--closing, and only hides once the opacity transition finishes.

This message fades out before it collapses. Try toggling repeatedly to see the difference.

<button
  data-automagica11y-toggle="#details"
  data-automagica11y-animate="target"
  data-automagica11y-target-class-open="panel--open"
  data-automagica11y-target-class-closed="panel--closing">
  Toggle details
</button>

Keyframe-driven toast

Here the target plays two iterations of a keyframe animation (`toast-out`) before the plugin hides it. Because the animation is on the target, `data-automagica11y-animate="target"` still applies.

Heads up

This notification collapses only after its animation is finished.

<button
  data-automagica11y-toggle="#toast"
  data-automagica11y-animate="target"
  data-automagica11y-target-class-open="toast--visible"
  data-automagica11y-target-class-closed="toast--closing">
  Toggle toast
</button>

Trigger-side animation

Sometimes the trigger itself owns the animation (e.g., glowing pill). Set data-automagica11y-animate="trigger" so the plugin watches the trigger element while the target still opens/closes as usual.

This inline region opens instantly, but the trigger glows while closing so users see visual confirmation.

<button
  data-automagica11y-toggle="#pill-panel"
  data-automagica11y-animate="trigger"
  data-automagica11y-trigger-class-open="pill--open"
  data-automagica11y-trigger-class-closed="pill--closing">
  Highlight block
</button>
